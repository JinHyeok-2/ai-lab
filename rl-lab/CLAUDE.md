# CLAUDE.md — rl-lab (RL 트레이딩 에이전트 개선 실험실)

## 목적

기존 `../rl/` 모듈의 PPO 트레이딩 에이전트를 체계적으로 개선하는 서브 프로젝트.
실험 → 검증 → 프로덕션(`../rl/`)에 반영하는 흐름.

### v5 핵심 개선점 (v4.1 대비)
- 에피소드 분할 (2000캔들) → 다수 학습 사이클
- 커리큘럼 러닝 (하락장 집중 → 자연분포)
- 보상 리스케일: pnl x 20 (v4.1: x 8)
- 피처 3개 추가: StochRSI, OBV 기울기, 변동성 레짐
- 관망 보상: upnl 기반 → 추세 정렬 기반

### 알려진 문제점
- 숏 편향 심함 (실거래 10/11건 숏)
- 실거래 테스트넷 승률 36% (4/11) — 백테스트 대비 저조
- 학습 데이터 기간에 따라 성능 차이 큼
- 보상 함수 패널티 과다 (cooldown, min_hold, 추세 역행 등)

## 디렉토리 구조

```
rl-lab/
├── CLAUDE.md          ← 이 파일
├── experiments/       ← 실험별 코드 (exp01_xxx.py ...)
├── data/              ← 실험용 데이터 (심볼링크 또는 복사)
├── models/            ← 실험 모델 저장
├── notebooks/         ← 분석 노트북
└── baseline.md        ← 베이스라인 성능 기록
```

## 실행 환경

- conda 환경: `ai-lab` (Python 3.12)
- 실행: `conda run -n ai-lab python <script>`
- 기존 rl/ 코드 import: `sys.path.insert(0, parent_dir)`

## 실험 규칙

1. 실험 파일명: `exp{번호}_{설명}.py` (예: `exp01_reward_shaping.py`)
2. 모델 저장: `models/exp{번호}/` 하위
3. 기존 `../rl/` 파일은 직접 수정하지 않음 (검증 후 반영)
4. 실험마다 baseline.md에 결과 기록

## 개선 후보 (우선순위)

1. **보상 함수 재설계** — 현재 패널티 과다, Sharpe ratio 기반 보상 검토
2. **피처 엔지니어링** — 오더북 임밸런스, 펀딩비, OI 변화율 추가
3. **멀티 타임프레임 입력** — 30m + 1h + 4h 동시 관측
4. **숏 편향 해소** — 양방향 균형 보상 설계
5. **앙상블** — 여러 모델 투표 방식
6. **하이퍼파라미터 튜닝** — Optuna 자동 탐색
